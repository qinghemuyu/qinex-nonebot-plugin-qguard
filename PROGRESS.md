# QGuard 开发进度

更新时间：2026-06-17

## 总体状态

文档内 v0.1 到 v0.5 的主体功能已经落地，当前继续按验收清单补边角能力、测试覆盖和 README 同步。

## 已完成

- 阶段 1：项目骨架、配置、PluginMetadata、NoneBot 加载。
- 阶段 2：数据库模型、仓库层、SQLite 自动建表和兼容迁移。
- 阶段 3：OneBot v11 `GroupOps` 适配器封装。
- 阶段 4：权限系统和审计日志。
- 阶段 5：基础群管命令。
- 阶段 6：群名片管理。
- 阶段 7：群名片锁、手动修复、事件兜底、自动巡检。
- 阶段 8：自动审核、规则、消息缓存、白名单保护。
- v0.3：广告检测、广告词库、刷屏检测、违规积分。
- v0.4：新人保护、入群审核。
- v0.5：群名锁、匿名锁、头衔、巡检、自动巡检。
- 追加能力：机器人消息自动撤回、角色管理、按用户/类型日志查询、最近消息查询、单条缓存消息查询。

## 本次推进

- 自动撤回新增消息分类：指令消息、聊天消息。
- `/管 自动撤回 90s` 继续设置撤回时间，`/管 自动撤回 分类 指令|聊天|全部|关闭` 设置撤回范围。
- QGuard 指令回复标记为指令消息；AI Core 售后/聊天回复标记为聊天消息。
- 默认只撤回指令消息，AI 聊天内容默认保留。

## 待继续核对

- 对更多命令补充直接命令层测试，目前已有服务层/核心逻辑测试，但命令解析层还可以继续加厚。
