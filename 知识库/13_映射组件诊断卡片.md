# 映射组件诊断卡片

这篇给智能问答用,专门处理“某个映射组件是不是坏了 / 为什么没反应 / 应该怎么配”。回答时先判断用户问的是**整个映射无输出**还是**某个组件的固有机制**。

如果是“任意键都摆好了但游戏里完全没有触摸”,再去查保存配置、输出模式、参考分辨率、管理员权限。

如果是某个组件的机制问题,不要套通用排查。先讲清组件怎么工作,再给正确配置方式。

## 自定义按键 / 普通按键 / 鼠标键 / 侧键

用户常见说法:

普通按键没反应,鼠标侧键怎么绑,滚轮上下能不能绑,一个键能不能点两个地方,我想走位时再点一个键。

机制:

自定义按键就是把一个键映射成屏幕上的一个或多个触点。它适合“按一下点屏幕”或“按住保持触摸”。鼠标左右键、侧键、滚轮上/下都可以作为触发键。

关键口径:

一、自定义按键可以和移动摇杆同时生效,前提是这个键不要占用 W/A/S/D 方向键。

二、一个键可以映射多个触点,但同一个键被很多组件占用时要看键冲突检查。

三、如果是所有自定义按键都没输出,再查保存配置、输出模式、参考分辨率、管理员权限。

正确做法:

想“一边走位一边点某个技能”,那个技能用独立自定义按键,不要用 W/A/S/D 方向键。

## 移动摇杆 / WASD / 走位

用户常见说法:

按住 W+A 再按 D 没反应,同时按两个方向键没用,斜向走位再按一个方向键不触发,WASD 组合键是不是 bug。

机制:

WASD 是一个移动摇杆 / 一个触点,不是 4 个独立按键。它会把当前按住的方向键合成成唯一一个方向,和真实手柄左摇杆一样,同一时刻只能指一个方向。

关键口径:

一、左右 A/D 互相抵消,前后 W/S 互相抵消。

二、按住 W+A 是左前斜向;这时再按 D,只是把 A 抵消,方向变成正前 W,不会额外触发动作。

三、这不是键冲突、不是没保存、不是输出模式不对。

正确做法:

如果想让某个动作在走位时独立点屏,用单独的自定义按键摆自己的触点,不要把这个动作塞进 W/A/S/D 方向键。

## 视角滑动区域 / 鼠标视角

用户常见说法:

鼠标动但视角不动,视角太飘,滑屏忽快忽慢,开镜后甩飞,鼠标模式和视角模式分不清。

机制:

视角滑动区域把鼠标移动转换成屏幕滑动,它不是点击按钮的组件。核心参数是灵敏度、加速度曲线、纯净模式和自动回中。

关键口径:

一、鼠标动但视角不动,先确认当前在视角模式,不是鼠标光标模式。默认 ALT 切换。

二、太飘先降灵敏度;忽快忽慢先把加速度曲线调平或全 1。

三、追求最稳定时用纯净模式;觉得卡顿再查回报率、异步、出帧。

四、如果所有视角都不动,再查输出模式、保存配置、管理员权限。

正确做法:

先让用户确认 ALT 模式,再调灵敏度和加速度,不要上来就让用户重装或改授权。

## 鼠标轮盘 / 环形技能盘

用户常见说法:

鼠标轮盘怎么用,按住 Tab 后怎么选技能,长按进轮盘没反应,滚一下鼠标滚轮能不能当轮盘。

机制:

鼠标轮盘组件是“短按=点击触发键触点,长按=锁鼠标进入轮盘滑选”。它适合环形技能盘或按住某键后用鼠标方向选择的场景。

关键口径:

一、鼠标轮盘不是物理滚轮上下滚动本身,物理滚轮是 WHEELUP / WHEELDOWN 这类瞬时键。

二、鼠标轮盘、移动摇杆、视角区域属于基础层专属,不要放进扩展层里找。

三、长按不进轮盘,先看长按触发时间和绑定键。

正确做法:

需要“滚一下切倍镜”,优先用手势滑动键绑定 WHEELUP / WHEELDOWN;需要“按住 Tab 进环形盘”,用鼠标轮盘组件。

## 开镜灵敏度 / ADS

用户常见说法:

开镜灵敏度没效果,右键开镜没反应,开镜后太快,开镜甩飞,ADS 是不是会自动点开镜。

机制:

开镜灵敏度 ADS 只负责“按住某键时临时降低视角灵敏度”,它本身不负责点屏幕开镜按钮。真正开镜的触点还要用自定义按键摆在游戏开镜按钮上。

关键口径:

一、ADS 不是开镜点击,它只是降灵敏。

二、右键开镜常见做法是:右键自定义按键点开镜按钮 + ADS 绑定右键降灵敏。

三、右键还能同时激活按键层,这是允许的叠加;键冲突提示可以按配置目的判断。

正确做法:

用户说“右键开镜没反应”,先问是开镜按钮没点到,还是开镜后视角没降灵敏。两个问题不是一回事。

## 多态交互 / 背包 / 商店 / 地图

用户常见说法:

按 B 打开背包后怎么点装备,背包模式退不出来,商店/地图模式怎么切,为什么打开后鼠标点不准。

机制:

多态交互是“按一下进入某种模式,再按一下或按退出键回去”。常用于背包、商店、地图。它通常需要用户能看见手机画面,也就是投屏或 vPointer,否则只能盲点。

关键口径:

一、多态交互不是单纯点一下按钮,它是一种模式切换。

二、背包/商店里要精准点装备,最好配合投屏或 vPointer。

三、退不出来先按原触发键或设置的退出键,不要先怀疑映射整体坏了。

正确做法:

用户问背包模式,优先确认有没有投屏或 vPointer,再讲打开 tap、关闭 tap 和再按退出。

## 轻触 / 长按双功能

用户常见说法:

一个键怎么短按换弹长按丢武器,轻触老是触成长按,长按不触发,同一个键能不能又普通映射又轻触长按。

机制:

轻触 / 长按把同一个键分成两个动作:短按触发轻触点,按住超过阈值触发长按点。

关键口径:

一、配了轻触 / 长按的键,不要再在普通按键里给同一个键配触点,容易打架。

二、误触多就调长按阈值,比如把阈值适当加大。

三、轻触点和长按点是两个不同落点,都要拾取。

正确做法:

用户说轻触长按不准,先看阈值和同键冲突,不要先让他查管理员权限。

## 手势滑动键

用户常见说法:

滚轮切倍镜怎么做,一个键划菜单,手势滑动不动,起点终点怎么设,为什么滚一下没连续滑。

机制:

手势滑动键是一个键触发一次从起点到终点的滑动。它适合切倍镜、滑条、菜单滑动。物理滚轮 WHEELUP / WHEELDOWN 可以触发一次手势。

关键口径:

一、手势必须有起点和终点,两个点都要放准。

二、滚轮是瞬时键,滚一下触发一次手势,不是一直按住。

三、手势放在哪个按键层,就只在那个层生效。

正确做法:

用户问滚轮切倍镜,优先建议 WHEELUP / WHEELDOWN 分别绑定两个相反方向的手势。

## 按键层 / 扩展层 / 载具层 / 法术栏

用户常见说法:

按住某键换一套键,载具层不生效,开镜层里滚轮才切倍镜,为什么层里看不到摇杆/视角,右键切层提示冲突。

机制:

按键层是按住或切换某个激活键时临时换一套键位。基础层是平时那套,扩展层是临时套上去的一套。

关键口径:

一、组件放进哪层,就只在那层生效。

二、移动摇杆、视角滑动、鼠标轮盘、ADS 属于基础层专属,进扩展层会锁住或不可改。

三、按住型层松开就回基础层;切换型层需要再按或退出。

四、右键这类可以“发触点 + 激活层/ADS”叠加,冲突检查提示不一定是错。

正确做法:

用户说某层不生效,先确认当前是在编辑那一层,激活键是否按住/切换成功,目标组件是不是放在对应层。

## 蓄力瞄准 / 手雷 / 弓

用户常见说法:

手雷按住怎么瞄准,蓄力时视角乱动,松手不扔,弓箭蓄力怎么配。

机制:

蓄力瞄准是按住触发键后,触点随鼠标拖动瞄准;松开抬起触点,相当于投掷或释放。蓄力期间视角不动。

关键口径:

一、蓄力不是普通长按触点,它会接管鼠标移动去调落点。

二、必须拾取起始触点,也就是手雷/弓/技能按钮的位置。

三、松手不扔,先看起始触点位置和触发键,再看游戏本身是否要求二段确认。

正确做法:

用户问手雷或弓,优先讲“按住起手、鼠标调落点、松手投出”,再让他调灵敏度。

## 连点 / Turbo

用户常见说法:

怎么自动连点,按住才连点还是开关连点,连点太快没反应,快速拾取怎么配。

机制:

连点组件按节拍反复点击某个触点,可以做连发、快速拾取。它和普通按住触点不是一回事。

关键口径:

一、连点要拾取连点触点,并设置间隔。

二、间隔太短可能被游戏吞掉或表现不稳定,先用保守间隔测试。

三、需要防检测时,保留一点随机半径或抖动。

正确做法:

用户问连点没效果,先确认触点位置、触发方式和间隔,不要直接让他拉到最快。

## 压枪 / 后坐力补偿

用户常见说法:

压枪怎么开,压枪没效果,仅开镜时压枪,压枪会不会封号,下拉强度怎么调。

机制:

压枪是在开火期间按时间给视角通道做下拉补偿,不是替你识别枪械或自动瞄准。压枪对反作弊最敏感。

关键口径:

一、压枪默认应保守开启,强度从小到大调。

二、如果开启“仅开镜时”,要确认 ADS 或开镜键状态确实触发。

三、必须提醒反作弊/封号风险,开启自行承担。

正确做法:

用户问压枪,先讲小强度试起,再讲风险。不要承诺安全、无封号、自动适配所有枪。

## 回答组件问题的通用模板

一、先判断是不是组件固有机制。

二、如果是机制问题,第一句直接给结论:“这个不是 bug,是某组件的工作方式。”

三、第二句讲正确用法。

四、只有用户说“所有键都没输出”,才问保存配置、输出模式、参考分辨率、管理员权限。
