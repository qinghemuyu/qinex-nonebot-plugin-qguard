# 术语别名与口语化问法

这篇用来把用户在群里的口语说法翻译成 QInEX 知识库里的正式说法。回答时先按这里归一化问题,再去对应文档里给排查步骤。

## 走位 / 摇杆 / WASD / 组合键 / 同时按 / 斜向

用户说“走位”“摇杆”“WASD”“方向键”“同时按两个键”“按住斜向再按一个没反应”“组合键不触发”“wa/wd 再按 d 没用”,几乎都指**移动摇杆(WASD 轮盘)组件**,不是泛泛的“按键没反应”。

回答要点(机制问题,不是 bug):

一、WASD 是**一个摇杆 / 一个触点**,把当前按住的方向键合成成**唯一一个方向**,同一刻只能指一个方向(和真实手柄左摇杆、和游戏内推摇杆一致)。

二、左右(A/D)互相抵消、前后(W/S)互相抵消。按住 W+A 再按 D = 把 A 抵消、方向变正前 W,所以“D 没反应”,属于正常。

三、**不要**对这类问题套“输出模式 / 管理员权限 / 保存配置”那套通用排查,治不了,还显得不专业。

四、想要某键在走位时独立点屏,改用单独的「自定义按键」摆自己的触点(能和摇杆同时触发)。

五、对应文档:04_配置面板-基础布键(移动摇杆)、10_排障与卡顿速查、FAQ问答对。

## 映射组件 / 组件机制 / 高级组件

用户说“组件是不是坏了”“这个功能没反应”“为什么不像我想的那样触发”,先判断是哪一个组件:

自定义按键、移动摇杆、视角滑动区域、鼠标轮盘、开镜灵敏度 ADS、多态交互、轻触/长按、手势滑动键、按键层、蓄力瞄准、连点、压枪。

回答要点:

一、如果是组件固有机制,直接讲机制和正确用法。

二、如果是所有键都完全没输出,才查保存配置、输出模式、参考分辨率、管理员权限。

三、每个组件的详细售后口径见 13_映射组件诊断卡片。

## 上位机 / PC端 / 电脑端

用户说的“上位机”“PC 端”“电脑端”“电脑版”“电脑程序”“最新版上位机”,一般就是指 Windows 上运行的 QInEX 映射软件,也就是打开后带配置面板的那套程序。

不要把“上位机”当成另一个产品。它通常对应这些知识分类:

一、基础选型:01_产品简介与选型。

二、S3 硬件接线:02_硬件模式(ESP32-S3)接线与上手。

三、免硬件数据线:03_免硬件模式(数据线ADB)。

四、布键和参数:04_配置面板-基础布键、05_高级映射(开镜背包载具手势蓄力)。

五、卡顿和没反应:10_排障与卡顿速查。

## 配置面板 / Web UI / 布键页面

用户说“配置页”“网页面板”“布键页面”“拖键页面”“QInEX 页面”,通常就是 QInEX 里的配置面板。常见问题是:

一、改完没点保存配置,游戏里不生效。

二、工作模式没选对,例如本来用 S3 却选成免硬件,或反过来。

三、参考分辨率和手机不一致。

四、程序没有管理员权限运行,输入被游戏或系统拦住。

五、配置面板打不开或空白时,先检查 WebView2 运行时和程序目录文件是否完整。

## 校准 / 触摸校准 / 坐标偏移 / 部分按键失效

用户说“校准映射之后部分按键失效”“校准后有些按键没反应”“点位偏了”“点不准”“只有边缘按键失效”“上半边/下半边/某一侧按键不灵”,通常不是授权或设备损坏,优先归到**坐标变换 / 参考分辨率 / 黑边比例 / 触摸校准**。

回答要点:

一、先分清是所有键都不输出,还是只有部分点位失效。所有键都不输出才查保存配置、输出模式、管理员权限。

二、只有部分键失效时,重点查参考分辨率是否等于手机实际分辨率,以及校准/黑边/旋转是否改过。

三、P4、采集卡、外接屏场景尤其要查触摸校准:采集卡分辨率填对,先自动计算黑边,不准再手调上下左右黑边(%)。

四、把失效触点拖到游戏按钮中心偏内一点测试。能恢复就说明触点坐标偏到按钮外或边缘了。

五、最后再查保存配置、配置档是否切错、键冲突检查、按键层是否激活。

六、对应文档:10_排障与卡顿速查#校准映射后部分按键失效, FAQ问答对#校准映射之后部分映射按键失效。

## 手机 APP / QInEScreen / vPointer

用户说“手机 APP”“QInEScreen”“手机端”“vPointer”,多半不是核心 S3 触摸映射本体,而是手机侧辅助能力:

一、投屏。

二、悬浮光标。

三、手机上配置 P4。

S3 硬件模式的基础键鼠映射不要求手机安装 APP。只有投屏、光标、手机配 P4 这类功能才需要按 09_QInEScreen手机APP 去回答。

## S3 板子 / 接收器 / 硬件版

用户说“S3 板子”“接收器”“硬件版”“小板子”,一般是 ESP32-S3 硬件模式。用户侧只需要知道:

一、电脑连 S3, S3 再连手机 OTG。

二、手机会把它识别成标准 USB 触摸屏。

三、觉得黏、慢半拍、拖枪不跟手,优先把 S3 板载页的出帧模式切到丝滑。

四、电脑端回报率先设 1000,发送模式选异步。

五、线材和 USB 口很重要,优先主板直出 USB 口,不要走前置面板、hub、劣质延长线。

## P4 / 单机版 / 不开电脑

用户说“P4”“单机版”“不开电脑”“板载配置页”,对应 P4 单机版。重点是:

一、键鼠直接插 P4,板子内部完成映射,不需要电脑常开。

二、配置可以用板子自带 WiFi 配置页,也可以用 QInEScreen 手机 APP。

三、P4 吃同一份 config.json,电脑端配好的键位可以导入 P4。

四、P4 卡在重启、连不上配置页时,优先看芯片版本和固件版本是否匹配。

五、P4 有按键显示但手机无触点时,先看是否激活、配置是否导入、手机是否识别触摸。

## 免硬件 / 数据线 / ADB / UHID

用户说“免硬件”“一根线”“数据线模式”“ADB 模式”“不要板子”,就是免硬件模式。回答重点:

一、手机要开启开发者选项和 USB 调试,连接电脑后允许 USB 调试弹窗。

二、QInEX 配置面板里工作模式选免硬件或 adb。

三、延迟高时,完整版优先开 USB 网络共享直连,再开自动刷新率。

四、线插主板后置 USB3 口,不要走前置面板、hub、劣质延长线。

五、追求极致跟手时,S3 或 P4 硬件路径更稳。

## ScreenHub / 投屏 / 控制模式

用户说“投屏”“ScreenHub”“电脑看手机画面”“控制模式”“点对点操控”,对应投屏功能。注意区分:

一、投屏画面卡,不一定等于触摸卡。

二、投屏卡优先降分辨率、码率、帧率,编码可试 H.265。

三、控制模式退不出来,按 Esc 或绑定的操控热键。

四、改投屏分辨率、帧率、编码后通常要点应用或重开投屏;码率才是热更新。

## 回报率 / polling rate / 发送频率

用户说“回报率”“Hz”“8000”“鼠标 8K”“发送频率”,回答时要强调:

一、不是越高越好。

二、太高会让电脑 CPU 跟不上,反而抖动、掉帧、一卡一卡、发热。

三、先用 1000,稳定后再试 2000。

四、一变卡就往回降。

五、8000 只适合高端 CPU 和真实 8000Hz 鼠标场景。

## 同步 / 异步发送

用户说“同步”“异步”“发送模式”,按这个口径回答:

一、觉得卡、顿、拖枪不跟手,优先用异步。

二、异步更跟手,极限下可能丢一两个中间移动帧,连续移动通常无感。

三、同步更像一帧不丢,但发送时可能阻塞,表现成偶尔顿挫。

## 出帧模式 / 丝滑 / 稳帧

用户说“出帧”“丝滑”“稳帧”“板载出帧”,通常是硬件侧输出节奏:

一、S3 觉得触摸点黏、慢半拍、拖枪不跟手,优先切丝滑。

二、P4 视角忽快忽慢或抖,可按 P4 文档检查稳帧或出帧档位。

三、出帧设置和电脑端回报率是两件事,要分开排查。

## 卡顿的口语说法

这些说法都应归到卡顿/跟手问题:

掉帧、丢帧、一卡一卡、卡卡的、一顿一顿、忽快忽慢、抖、飘、慢半拍、拖枪不跟手、触摸点黏、滑屏不顺、视角不顺、感觉卡帧。

优先回答路径:

一、先问清卡的是 PC 端触摸、手机游戏画面,还是投屏画面。

二、PC 端触摸卡:回报率 1000,发送模式异步,关录屏/直播/杀毒实时扫描。

三、S3 触摸黏:板载页出帧模式切丝滑。

四、免硬件卡:USB 网络共享直连、自动刷新率、主板后置 USB3。

五、手机端卡:关省电、清后台、降画质、注意散热。

## 没反应的口语说法

这些说法都应归到不生效/无触摸问题:

没反应、不生效、按键没用、按了不动、没有触点、无触点、游戏里没用、保存了没效果、触摸不出来、键鼠不输出。

优先回答路径:

一、是否点了保存配置。

二、工作模式是否选对:S3 / 免硬件 / P4。

三、输出模式是否是触摸屏。

四、参考分辨率是否和手机一致。

五、是否在游戏内测试,且 QInEX 以管理员权限运行。

六、如果是 P4,确认配置已导入并重启生效;如果是 S3,确认手机识别到 USB 触摸屏。

## 上位机打不开 / 配置面板打不开

用户说“上位机打不开”“配置面板空白”“布键页面不出来”“启动闪退”,优先按低风险步骤回答:

一、右键以管理员身份运行 QInEX。

二、确认解压完整,不要只单独拿 QInEX.exe 出来运行,程序目录里的配套文件要放在一起。

三、如果提示缺少 WebView2 或配置面板空白,安装 Microsoft Edge WebView2 运行时后再试。

四、杀毒软件或系统防护误拦时,先恢复文件并加信任,再重新解压运行。

五、仍然不行时,让用户发启动提示截图或控制台最后几行日志,注意遮住隐私信息。

## 回答不了时该补什么信息

如果知识库命中不够,不要硬猜。让用户补最关键的 1 到 2 项:

一、当前用的是 S3、免硬件还是 P4。

二、问题发生在 PC 端触摸、手机游戏画面,还是投屏画面。

三、回报率是多少,发送模式是同步还是异步。

四、是否管理员权限运行,是否点过保存配置。

五、线材和 USB 口:主板后置、前置、hub、延长线。

六、手机是否开省电、是否发热、游戏画质帧率设置。

七、截图可以发,但不要发完整授权码、订单号、密钥、手机号、付款截图。

## 智能客服 / 问诊 / 缺口 / 补知识

用户说“机器人怎么用”“智能客服怎么问”“售后机器人”“问诊”“为什么先问我模式”“为什么不直接回答”“未命中”“知识缺口”“缺口看板”“补知识”“怎么让主人补知识”“连续对话”“解决了会记录吗”,归到智能客服与售后闭环。

对应 skill:`qinex_ai_support`。

优先回答路径:

一、普通用户问问题:推荐 @机器人、`/求助 问题描述`、`/售后 问题描述`、`/不会用 功能名称`。

二、问题太短时:解释“卡/没反应”太泛,机器人会先问 S3/ADB/P4、问题段落和全部/部分失效,这是为了避免乱猜。

三、连续跟进:同一用户说“还是不行”“没效果”“还卡”会继续上一轮;说“新问题/换个问题”会重置。

四、解决反馈:用户说“好了/解决了/可以了”,机器人会记录这条排查路线有效。

五、主人补知识:收到未命中 N 编号后,用 `/客服 补知识 N000001 答案内容`。答案只写公开使用步骤,不写授权码、密钥、算法或源码。

六、看缺口:用 `/客服 缺口`,优先补未解决次数和未命中次数高的问题。

七、群范围:如果补了知识仍不命中,先查 `/知识 范围`,智能客服相关分类是 `14_智能客服与售后闭环`,skill 是 `qinex_ai_support`。
