# P4 双模式:单机模式 / 上位机模式

P4 不是只有"单机版"。用户说 P4 时,先分清他当前跑的是哪种模式:

- **P4 单机模式 / Mode A**:键鼠直接插 P4,板子内部完成映射,再从 H2 USB-C 输出触摸给手机。玩的时候不需要电脑常驻。
- **P4 上位机模式 / Mode B**:电脑运行 QInEX 上位机负责映射,P4 作为下位机桥接硬件,把电脑发来的触摸帧用 USB HS HID 输出给手机。它更像"S3 硬件模式的 P4 高速版"。

判断口诀:

- 用户说"键鼠插板子、不想开电脑、板载配置页、手机配 P4" → 多半是 **P4 单机模式**。
- 用户说"电脑上位机、COM 串口、硬件/serial、8000Hz、P4 当 S3 用" → 多半是 **P4 上位机模式**。
- 用户只说"P4 卡/没反应"时,先追问一句:"你现在是键鼠直插 P4 单机,还是电脑 QInEX 上位机连 P4?"

## P4 单机模式 / Mode A

链路:

键鼠 → P4 USB-A → 板内映射 → H2 USB-C → 手机触摸

特点:

- 不需要电脑常驻,适合随身带、固定配置后直接玩。
- 配置入口是板子热点 `QInEX-P4-Config` / 密码 `12345678`,浏览器打开 `http://192.168.4.1/`。
- 也可以用 QInEScreen 手机 APP 的 P4 映射配置功能直接编辑、推送配置。
- 吃同一份 `config.json`:电脑端/配置网站导出的键位配置可以导入 P4。
- 板内映射会复刻 PC 上位机的主要映射字段:基础按键、WASD 摇杆、视角、鼠标轮盘、滚轮、按键层、ADS、连点、压枪、蓄力、手势等。
- PC 专属字段会被板子忽略,比如电脑端投屏、PC 授权、部分 Windows 行为。

### 单机模式快速上手

1. 跳线 **H3 = HOST**。
2. 键鼠插 P4 的 **USB-A** 口。
3. 手机插 **H2 USB-C** 口。
4. 手机或电脑连接热点 `QInEX-P4-Config`,打开 `http://192.168.4.1/`。
5. 在激活页确认已激活;未激活时不会输出触摸。
6. 导入从 PC/配置网站导出的 `config.json`,导入后按页面提示重启。
7. 进游戏测试;手感不对时优先调板载页的实时灵敏度、旋转、触摸校准、出帧档位。

### 单机模式板载配置页

状态:

- 看固件版本、MAC、内存、USB 是否挂载、是否已导入配置、激活状态。
- 输入监控能看到键鼠是否被识别。设备栏空 = 键鼠没识别;有按键显示但手机无触点 = 通常是激活/配置/触摸输出问题。

激活:

- 未激活不输出触摸。
- 激活用的 MAC 必须从 P4 板子专属配置页复制,不要用电脑网卡、路由器列表或手机蓝牙里的 MAC。
- 涉及授权码、订单、换绑、异常,不要在群里发完整隐私信息。

映射:

- 导入配置文件:选 `config.json` 后写入板子,一般需要重启后生效。
- 灵敏度(实时):不用重导配置,拖动即时生效并保存。
- 出帧档位:默认/高档/超高/极速,越高越跟手但负载更高。单机模式最高按 1000Hz 档理解。
- 稳定出帧:视角忽快忽慢时可以开。
- 屏幕旋转:横竖屏方向不对时调。
- 触摸校准:采集卡/外接屏导致黑边或偏移时用。先自动计算,不准再手调上下左右黑边。

设备:

- HID 设备信息:改手机看到的 VID/PID/厂商名/产品名,改后需要重启重新枚举。
- kmboxNET:可以把 P4 当网络盒子用。
- vPointer:手机显示可见光标,用于背包/菜单/鼠标模式;需要 QInEScreen/vPointer 同网。

系统:

- 固件升级:上传官方 OTA 包,升级中不要断电。
- 调试模式:键鼠不识别或映射异常时打开,把日志反馈给开发者。
- 恢复出厂会清空配置/灵敏度/vPointer/WiFi 等设置,授权通常保留。

## P4 上位机模式 / Mode B

链路:

PC QInEX 上位机 → UART0 / H1 CH343 串口 → P4 → USB HS HID → 手机触摸

特点:

- 需要电脑常驻运行 QInEX。
- P4 在链路里相当于高速下位机/触摸桥,角色接近 S3,但 P4 走 USB HS,设备侧最高可按 8000Hz 设计。
- 配置和映射在 PC 上位机里编辑,工作模式选 **硬件 / serial**。
- 上位机通过串口连接 P4;P4 会自报 `ESP32-P4` 和 `MODE=B`,上位机识别后会启用 P4 的 ping/RTT 检测。
- Mode B 的板载配置页主要用于设备伪装、输出模式、出帧策略、授权、OTA、重启,不是主要布键入口。

### 上位机模式怎么用

1. P4 需要刷入或 OTA 切到 **Mode B / 上位机模式** 固件。
2. 手机接 P4 的触摸输出口。
3. PC 连接 P4 的 H1 CH343 串口。
4. 打开 QInEX 上位机,工作模式选 **硬件 / serial**。
5. 选对 COM 口,保存配置并重启上位机连接。
6. 需要高回报率时,在上位机里调轮询率;8000Hz 不一定越高越好,普通电脑先从 1000 或 2000 试。

### Mode B 配置页

进配置页的思路:

- 上位机可以通过 `CONFIG` 命令让板子下次进入 WiFi 配置页。
- 配置页热点同样是 `QInEX-P4-Config` / `12345678`,浏览器打开 `http://192.168.4.1/`。

配置页能做:

- 看固件/模式/MAC/授权状态。
- 修改 VID/PID/厂商名/产品名。
- 切输出模式:触摸屏 / 手柄。
- 切出帧策略:稳定 / 丝滑。
- 填 License 激活。
- 上传 OTA 包升级或切回 Mode A。
- 重启设备。

## Mode A 和 Mode B 怎么切换

P4 的单机模式和上位机模式是两套固件,可以通过配置页 OTA 切换:

- **Mode A 单机 → Mode B 上位机**:在单机模式配置页的固件升级里,上传 Mode B 的 OTA 包。
- **Mode B 上位机 → Mode A 单机**:在上位机模式配置页的固件升级里,上传 Mode A 的 OTA 包。

注意:

- 普通用户只需要拿官方给的 OTA 包上传,不要自己混用来源不明的 bin。
- P4 有不同芯片版本:老芯片常说 eco2 / v1.x,新芯片常说 eco6+ / v3.x。固件包也分普通版和 v3 版。
- 版本不匹配时可能启动异常或被升级页拒绝;烧错后反复重启、连不上配置页,优先换对应芯片版本的包。
- 已加密/量产板不要让用户自己乱刷整片镜像,普通售后只引导上传官方 OTA 包。

## P4 单机 / P4 上位机 / S3 怎么选

| 场景 | 推荐 |
|---|---|
| 不想开电脑,键鼠直接插板子玩 | P4 单机模式 |
| 要在电脑上实时编辑配置、投屏、用上位机完整能力 | P4 上位机模式或 S3 硬件模式 |
| 追求高速 USB HS 输出、想让 P4 当高速触摸桥 | P4 上位机模式 |
| 已有稳定 S3 硬件链路,不需要 P4 单机 | 继续用 S3 |
| 手机上直接配 P4 键位 | QInEScreen + P4 单机模式 |

## 常见排障

### P4 单机键鼠不识别

看板载页「输入监控」。如果识别设备为空,优先换高速游戏键鼠或有线键鼠;部分普通办公键鼠、无线 dongle、低速/全速设备可能不兼容。确认键鼠插 USB-A,手机插 H2 USB-C,跳线 H3=HOST。

### P4 单机有按键显示,手机没触点

先看激活状态。未激活不会输出触摸。再看是否已导入配置、导入后有没有重启、手机是否识别到 USB 触摸屏、触摸校准是否把点压偏了。

### P4 上位机连接不上

先确认当前固件是 Mode B。再查 PC 上位机是否选了硬件/serial、COM 口是否正确、串口是否被烧录工具或别的软件占用。上位机日志里如果能看到 `ESP32-P4` / `MODE=B`,说明板子已经被识别到。

### P4 上位机 8000Hz 反而卡

8000Hz 对电脑处理、串口发送、鼠标回报率都有要求,不是越高越稳。普通机器先用 1000 或 2000,稳定后再试更高。出现掉帧、发烫、视角忽快忽慢时先降回报率。

### 触摸校准后部分按键失效

这多半是黑边/参考分辨率/坐标压缩问题,不是授权坏。P4、采集卡、外接屏场景尤其要检查触摸校准:先填采集卡分辨率自动计算,再手调上下左右黑边,把失效触点拖到按钮中心偏内侧测试。

### 手机出现幽灵触点

常见原因是视角自动回中时间设成 0,板子一直按住视角触点。把自动回中时间设为大于 0,比如 800ms,空闲后自动抬起。

## 回答口径

用户问 P4 时不要直接默认"单机版"。先判断:

- "不想开电脑 / 键鼠插板子 / 板载配置页 / QInEScreen 配 P4" → 按 Mode A 单机回答。
- "上位机 / 串口 / COM / 硬件模式 / 8000Hz / 当 S3 用" → 按 Mode B 上位机回答。
- 没说清就追问一句当前模式,再给低风险检查:单机看板载输入监控和激活;上位机看 Mode B 固件、COM 口、硬件/serial。
