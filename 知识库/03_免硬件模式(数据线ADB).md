# 免硬件模式(一根数据线)

免硬件模式**不需要 S3 接收器**,只用一根 USB 数据线把电脑和手机连起来就能用。代价是延迟天花板比硬件版略高,追求极致跟手仍建议上硬件版。

> 原理(对外可讲版):电脑通过 USB 调试通道,在手机上临时拉起一个小巧的常驻进程来接收触控、转成系统级触摸,**用完即走、不常驻、不装 App、不要 ROOT**。

## 需要的东西

- 电脑(Windows x64)+ QInEX 程序
- 一根 USB 数据线(电脑↔手机)
- 手机开启**开发者选项 + USB 调试**

## 上手步骤

1. 手机开启 **开发者选项 → USB 调试**。
2. 一根 USB 线连电脑和手机,手机上弹出"允许 USB 调试"时点**允许**(可勾选始终允许)。
3. 运行 QInEX 程序。
4. 配置面板**工作模式选「免硬件 / adb」**。
5. 拖拽布键、设参考分辨率、**保存配置**。
6. 进游戏,按 **ALT** 切换模式。

## 确认手机被识别

如果连不上,先确认电脑能认到手机:在电脑命令行(cmd)运行 `adb devices`,能列出你的设备就说明通道正常。然后拔插 USB、重启 QInEX 再试。配置面板的「设备激活 & 状态」会给出具体错误提示,按提示处理。

## 降延迟:USB 网络共享(完整版)

免硬件模式默认走端口转发,延迟略高。**完整版**可在「高级参数 → ADB 配置」勾选 **USB 网络共享直连**:

1. 手机打开 **USB 网络共享**(系统设置里搜"USB 网络共享 / USB tethering")。
2. 程序自动走直连通道(更稳、延迟更低),失败会自动回退普通转发。

再配合打开 **自动刷新率**(让出帧节拍跟随手机刷新率),延迟体验会明显改善。

## 常见卡顿 / 连不上

- **特别卡 / 延迟高**:开 USB 网络共享直连 + 自动刷新率;数据线**插主板后置 USB3 口**,别走前置面板/hub/劣质延长线。
- **连不上**:`adb devices` 确认识别 → 拔插重启 → 看面板错误提示。
- **想要极致跟手**:免硬件本质是软件路径,延迟天花板高于硬件,真要极致就上 S3 / P4 硬件版。
